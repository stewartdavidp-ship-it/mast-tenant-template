# Enforcement & Guardrails — keeping the standard maintained

The redesign is **not a one-time migration**. Without enforcement, new development drifts straight back to the fragmentation the audit measured. These standards (docs 02–15) are kept true by **two complementary guards**, both wired into the repo's existing CI gate.

## Guard 1 — Enforcement by construction (the engines)
The strongest guard is architectural: CRUD UI is **generated by `mastEntity` from a schema** (doc 15), so a feature physically *cannot* ship an off-standard list, slide-out, dirty-handling, or import/export — there's no hand-written UI to diverge. Reports go through `mastReport`, media through `mast-media`, etc. New work = write a schema, not UI. This covers the ~30 CRUD modules + reports. **Prefer adding to / extending an engine over hand-writing a surface.**

## Guard 2 — The conformance ratchet (the hand-written edges)
For the edges engines don't generate (builders, bespoke bits, the shell), a CI linter prevents regression: **`scripts/lint-ux-standards.js`**.

- **Ratchet model:** it baselines today's per-module violation counts (`scripts/ux-standards-baseline.json`) and **fails any PR that increases any count**. New module files are born at **0**. As a module converts, its counts drop and `--update` re-locks the lower baseline — the numbers can only ever go **down**.
- **Tracked metrics** (high-signal, from doc 01): `nativeDialogs` (`window.confirm/alert/prompt` → `mastConfirm`), `rogueOverlays` (`position:fixed` → `mastSlideOut`/`openModal`), `translateX` (custom drawers → shared slide-out), `hardcodedHex` (→ `var(--…)` tokens, both-mode).
- **Current baseline: 42 files, 2806 violations** — this is the live debt KPI. End state: all-zero, then flip the linter to hard-zero (no baseline).
- **Run:** `node scripts/lint-ux-standards.js` (check; CI) · `--update` (re-lock lower baseline after a conversion).

## Where the gates run
- **CI (`.github/workflows/lint.yml`) — authoritative, un-skippable** (per the repo's Build-5 model that moved enforcement off the opt-in pre-commit hook). The ratchet runs alongside the existing blocking lints:
  - `lint-design-tokens.js` (font-scale **blocking**; hex advisory) · `lint-mastdb.js` · `lint-rbac.js` · `lint-module-info.js` · `lint-cache-bust.js` · **`lint-ux-standards.js` (new)**.
- **Pre-commit (`.githooks/pre-commit`) — optional/local** fast feedback; CI is the real gate.
- **Both-mode visual gate:** `scripts/capture-modes.js` (Playwright) on every redesign PR — dark + light screenshots attached; nothing disappears on theme flip (doc 02 §4b). Not a pure lint; a required PR artifact.

## Keeping the *standard itself* honest
- **The spec is the source of truth:** `docs/ux-audit/00–16`. Changes to a standard go through a PR that updates the doc — not silent divergence in code.
- **Suggested:** `CODEOWNERS` on `docs/ux-audit/` + `shared/mast-*.js` so engine/standard changes get review; a PR-template checkbox "conforms to docs/ux-audit (or updates it + ratchet)".
- **The rubric is re-runnable:** re-grade a touched module against `00-rubric.md`; CONTROL-PLANE tracks scores so regressions are visible.

## Roadmap to hard enforcement
1. **Now:** ratchet live in CI (drift blocked; debt frozen at 2806).
2. **Per conversion:** counts drop → `--update` locks them lower; converted modules pin at 0.
3. **As engines mature:** add targeted rules (e.g., "new `app/modules/*-v2.js` must `MastEntity.define(...)`"; "no raw `<input type=file>` outside `mast-media`").
4. **End state:** baseline all-zero → replace the ratchet with **hard-zero** lints (any violation fails), and the standard is self-maintaining: engines generate conformant UI, CI rejects the rest.
